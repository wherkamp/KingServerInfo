CustomArguments.yml

How to configure CustomArguments.yml in KingServerInfo

Rules for making a custom argument

  1. Every Arument Must Have a Description
  2. Arguments must be real characters

This is the default CustomArguments.yml


#Everyone New Arguments must be one word long and will work using equalsIgnoreCase()
#Permissions will be kingserverinfo.command.{argument}
#PlaceholderAPI placeholders are allowed
#You cannot overwrite a hardcoded Argument Such as <command> staff

Arguments:
   QA:
      Alias:
         - qanda
      Player-Message:
         - 'No we do not sell pandas, %player_displayname%'
      Broudcast-Message:
         - '%player_displayname% wondered if we sold pandas'
      Console-Commands:
         - 'give %player_displayname% Diamond 1'
      Player-Commands:
         - 'help'
      Description: 'This will answers some basic questions.'

List of tasks you can add inside of a custom argument


  1. Player-Message: will send a message to the player
  2. Player-Commands: will make the player run a command
  3. Broudcast-Message: Will broudcast a message to the chat
  4. Console-COmmands: Will have the console run a commmand
  5. Alias will add alias's to the argument
  6. Any PlaceholderAPI placeholder?

This is the format of a new argument

            hello: <--- The name
               Description: This will be the description
               <--- Any other jobs such at Alias or Player-Message
